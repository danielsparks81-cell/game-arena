# HeroScape — Spec: 2.5D Isometric Board Renderer

> A renderer upgrade, NOT a rules change. Replaces the flat top-down SVG hex
> board with a 2.5D isometric view: stacked hex-prism terrain (using the
> per-cell height the engine already stores) and figures as upright standee
> sprites. The engine (state, hex coords, heights, legal-move/attack helpers,
> projection) is untouched — only `HeroScapeBoard.tsx`'s board SVG changes.

## Goal

Get the "VirtualScape / HeroScape Online" look: terrain reads as stacked 3D
hex columns at their real elevations, and figures stand on top as little
upright sprites instead of flat discs. Stay in SVG (no 3D engine) — it's a
projection + draw-order change over data we already have.

## Coordinate model

The engine cells are axial hexes `(q, r)` with `height` (levels). board.ts
already has `hexToPixel` (flat pointy-top). Add an ISOMETRIC projection:

```
// Per-cell screen anchor (TOP-face center) in unit space, before scaling:
isoX = (q - r) * ISO_DX
isoY = (q + r) * ISO_DY - height * LEVEL_H
```
- Use a dimetric hex layout (pointy-top hexes squashed vertically ~0.55 so they
  read as ground). `ISO_DX`/`ISO_DY` set the tile footprint; `LEVEL_H` is the
  pixel rise per elevation level (the "column" depth).
- Keep the existing per-viewer 180° orientation flip (so my start zone is at the
  bottom-front): apply it to `(q, r)` BEFORE projecting (negate/rotate around the
  map center), exactly as the flat renderer flips `toScreen`.

## Drawing a tile (hex prism)

For each cell, draw, back-to-front:
1. **Side faces** (the column): for the front-facing edges of the top hexagon,
   draw a quad dropping straight down by `height * LEVEL_H` to the base, filled
   with a DARKER shade of the terrain color (left/right faces slightly different
   shades for form). Only the 2–3 front/visible edges need walls; skip the back
   ones (hidden). At height 0/1 the column is short; tall tiles read as cliffs.
2. **Top face**: the squashed hexagon, filled by terrain (grass/rock/sand/water)
   — reuse the existing terrain palette but apply it to the iso top hex. Water
   gets a flat blue top, no tall column.
3. The height number badge can stay (small, on the top face corner) or be
   dropped now that elevation is visually obvious — keep it, smaller.

**Draw order (painter's algorithm):** sort cells by `(r + q)` ascending then by
`height` ascending, and within the same footprint draw the column then the top.
Figures draw after their tile. This makes nearer/taller things overlap farther
ones correctly. (No z-buffer; the sort is enough for a hex field.)

## Figures — standee sprites

Replace the colored disc with an upright **standee** anchored at the tile's
top-face center, lifted to the tile's elevation:
- A small **base ellipse** in the player's accent color (so ownership is clear),
  drawn flat on the top face.
- A **billboard sprite**: the figure's portrait (see Assets) drawn as an upright
  image standing on the base, sized ~1.4× the tile width tall, bottom-anchored at
  the base center so it "stands" on the tile. Slight drop shadow ellipse under it.
- Overlays unchanged in meaning, re-anchored to the standee: order markers above,
  wound pip, the activation ring (green=can move / amber=can attack / dim=done),
  selection highlight, target ring. Squad figures keep their index.
- Taller-Height units may render a touch bigger (optional polish; not required).

## Assets — figure portraits

Crop each card's figure ILLUSTRATION region into a standee sprite at
`public/heroscape/figures/<cardId>.jpg` (16 files). The illustration sits in a
consistent upper-left region of every card; a fixed fractional crop
(~x:[0.04,0.46], y:[0.16,0.64] of the card) captures it. Generated by a script
in heroscape-extract (same pipeline as the card crops). Accept the painted
background for v1 (the standee frame/base hides the seams); clean cut-outs are a
later polish. CardArt-style `onError` fallback → if a figure sprite is missing,
fall back to the current colored disc + letter so nothing breaks.

## Interaction — keep it keyed by hex

All engine integration stays identical; only hit-testing changes:
- Each tile's TOP-face hexagon is the click target (an SVG `<polygon>` with the
  existing `onClick={() => clickHex(key)}`). Compute the iso top-hex points; the
  click logic (select / move / attack) is unchanged.
- **Legal-move / attack highlights:** recolor / outline the iso TOP face (green
  reachable, red-ringed targets) — same `legalDestinations`/`legalTargets` sets,
  just drawn on the iso top hex.
- **Glyphs:** badge on the iso top face (lit when occupied), as now.
- Start-zone tint, the per-viewer flip, the placement "click a figure then a
  glowing hex" flow, Grapple highlights — all preserved, re-projected.
- Figures are also clickable (select mine / target enemy) via the standee.

## Scope

IN (first iso slice): iso terrain prisms with elevation; figure standees with
portrait sprites + colored base + all existing overlays; iso hit-testing +
highlights + glyphs + markers; per-viewer flip; everything else (cards, log,
right rail, dice overlay, hover popovers) unchanged.

OUT (later): movement/attack tween animations (figures snap for now); an
orbit/zoom camera (fixed iso angle); true 3D models; clean alpha-cut figure
sprites; water animation. Leave `// iso polish:` markers where these would hook.

## Implementation notes

- Put the iso math (project, top-hex points, side-face quads, draw-order sort,
  inverse-free hit-test via the polygons) in board.ts as pure helpers with unit
  tests (anchor positions, ordering), mirroring the flat helpers — so the engine
  and board read one source.
- The board SVG `viewBox` must grow to fit the taller iso layout (columns add
  vertical extent); compute bounds from all projected tile tops AND column bases.
- Keep it a drop-in: same props, same callbacks, same `state`. A feature flag or
  a straight replacement is fine — straight replacement is simplest.

## Verify + ship

board.test.ts (iso helpers) + tsc + vitest (heroscape, 2×) + build. The map must
render at its real elevations (the Knoll's hill reads as a hill), figures stand
on top, clicks land on the right hex, highlights/markers/glyphs all show.
HeroQuest's 38 pre-existing failures stay out of scope.
